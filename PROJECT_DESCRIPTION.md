# دستیار هوشمند رزرو هتل تهران

## خلاصه

این پروژه یک **سیستم گفتگومحور (Conversational AI)** برای کمک به کاربران در **انتخاب و پیشنهاد هتل در تهران** است. کاربر به‌صورت طبیعی و به زبان فارسی نیازهای خود را بیان می‌کند — مانند منطقه مورد نظر، ستاره هتل، امکانات، نزدیکی به آدرس خاص، یا نوع سفر — و سیستم پس از جمع‌آوری اطلاعات کافی، چند هتل مناسب را پیشنهاد می‌دهد.

پروژه ترکیبی از **پردازش زبان طبیعی (NLP)**، **جستجوی برداری (Vector Search)**، **بازرتبه‌بندی با مدل زبانی (LLM Reranking)** و **رابط کاربری وب** است و روی داده‌های واقعی هتل‌های تهران پیاده‌سازی شده است.

---

## مسئله و اهداف

### مسئله
انتخاب هتل در شهر بزرگی مانند تهران، با صدها گزینه و معیارهای متنوع (موقعیت، امکانات، کیفیت، بودجه)، برای کاربر دشوار و زمان‌بر است. فرم‌های سنتی جستجو معمولاً فیلترهای ثابت دارند و نمی‌توانند ترجیحات پیچیده و زبانی کاربر را به‌خوبی درک کنند.

### اهداف پروژه
1. درک نیاز کاربر از طریق **گفتگوی طبیعی فارسی**
2. جستجوی هوشمند در پایگاه داده هتل‌ها با ترکیب **فیلتر متادیتا** و **جستجوی معنایی**
3. انتخاب نهایی بهترین هتل‌ها با **بازرتبه‌بندی LLM**
4. ارائه نتایج در یک **رابط چت کاربرپسند** با کارت هتل و جزئیات کامل

## خط لوله داده (Data Pipeline)

داده‌های خام هتل‌ها از API علی‌بابا جمع‌آوری و در چند مرحله پردازش شده‌اند:

| مرحله | اسکریپت | توضیح |
|-------|---------|-------|
| ۱ | `1_fetch_hotel_info_by_hotel_id.py` | دریافت اطلاعات هتل از API |
| ۲ | `2_clean_hotel_html.py` | پاک‌سازی HTML از فیلدهای متنی |
| ۳ | `3_add_tehran_zone.py` | افزودن منطقه تهران (شمال/جنوب/شرق/غرب/مرکز) از روی مختصات |
| ۴ | `4_add_facilities_aggregate.py` | تجمیع امکانات هتل در یک فیلد فارسی |
| ۵ | `5_embed_hotels_chroma.py` | embed کردن فیلدها و ذخیره در ChromaDB |

**مجموع داده:** ۱۳۳ فایل JSON هتل در `data/Tehran_New/`

**فیلدهای embed شده در ChromaDB:**
- `title_fa` — نام هتل
- `address` — آدرس
- `description_fa` — توضیحات
- `facilitiesAggregate` — امکانات

**متادیتای ذخیره‌شده (بدون embed):** `hotel_id`, `star`, `tehran_zone`, `field`

## روش جستجو (Hybrid Search)

سیستم از یک **جستجوی ترکیبی (Hybrid Retrieval)** استفاده می‌کند:

1. **فیلتر ساخت‌یافته:** منطقه تهران و ستاره هتل به‌عنوان فیلتر متادیتا در ChromaDB اعمال می‌شود.
2. **جستجوی برداری per-field:** برای هر ترجیح کاربر (نام هتل، امکانات، آدرس، توضیح کلی) جستجوی معنایی روی فیلد مرتبط انجام می‌شود و از هر فیلد تا ۴ نتیجه برتر گرفته می‌شود.
3. **ادغام نتایج:** شناسه‌های هتل یکتا می‌شوند.
4. **بازرتبه‌بندی LLM:** مدل زبانی با درک زمینه گفتگو، هتل‌های candidate را ارزیابی و حداکثر **۵ هتل** نهایی را انتخاب می‌کند.

این رویکرد نسبت به جستجوی کلمه‌کلیدی ساده، **درک معنایی** ترجیحات فارسی (مثلاً «هتل آرام برای خانواده نزد مترو») را بهتر پشتیبانی می‌کند.

---

## عامل گفتگومحور (Conversational Agent)

عامل با **LangGraph** پیاده‌سازی شده و شامل:

- **مدل زبانی:** OpenAI GPT (قابل تنظیم از `.env`)
- **حافظه سشن:** `MemorySaver` با `thread_id` برای ادامه گفتگو در یک سشن
- **ابزار (Tool):** `find_hotel` — فراخوانی جستجو هنگام کافی بودن اطلاعات
- **محدوده دقیق (Scope):** عامل فقط درباره هتل تهران پاسخ می‌دهد و سوالات خارج از موضوع را رد می‌کند

جریان گفتگو: عامل ابتدا سوالات کوتاه می‌پرسد (منطقه، ستاره، امکانات، نوع سفر)، سپس با فراخوانی ابزار، هتل‌ها را جستجو و به فارسی توضیح می‌دهد.

---

## رابط کاربری

رابط با **Streamlit** ساخته شده و ویژگی‌های آن:

- چت دوطرفه با چیدمان RTL (کاربر راست، دستیار چپ)
- نمایش **کارت هتل** با تصویر، نام، ستاره و منطقه در گرید ثابت سه‌ستونه
- دکمه شیشه‌ای «مشاهده جزئیات» روی هر کارت
- **پاپ‌آپ جزئیات** (`st.dialog`) با آدرس، امکانات، توضیحات و گالری تصاویر
- پشتیبانی از چند بار پیشنهاد هتل در یک گفتگو بدون به‌هم‌ریختن چیدمان
- دکمه «شروع گفتگوی جدید» برای سشن جدید

## نحوه اجرا

### پیش‌نیازها
- Python 3.10+
- کلید OpenAI در فایل `.env`:

```env
OPENAI_API_KEY=your_key
EMBEDDING_MODEL=text-embedding-3-large
GENERATION_MODEL=gpt-4o
```

### نصب

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### embed داده‌ها (یک‌بار)

```bash
python 5_embed_hotels_chroma.py
```

### اجرای اپلیکیشن

```bash
streamlit run run_app.py
```

مرورگر: `http://localhost:8501`

## نوآوری‌ها و نکات آموزشی

1. **ترکیب RAG و Agent:** به‌جای پاسخ مستقیم از مدل، عامل ابزار جستجو را فراخوانی می‌کند و پاسخ بر اساس داده واقعی است.
2. **جستجوی چندفیلدی:** embed جداگانه برای نام، آدرس، توضیحات و امکانات امکان تطابق معنایی دقیق‌تر را فراهم می‌کند.
3. **Reranking با LLM:** پس از retrieval، مدل زبانی کیفیت تطابق با نیاز کاربر را ارزیابی می‌کند.
4. **پردازش داده فارسی:** پاک‌سازی HTML، منطقه‌بندی جغرافیایی تهران، و تجمیع امکانات برای جستجوی بهتر.
5. **UX گفتگومحور:** کاربر نیازی به پر کردن فرم پیچیده ندارد؛ سیستم به‌صورت تعاملی اطلاعات را جمع می‌کند.
